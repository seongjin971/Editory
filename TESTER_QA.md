# Editory 테스터 QA 메모

이 문서는 작가 테스터에게 링크를 전달할 때 함께 보는 최소 체크리스트입니다.

## 로그인

- 이메일: 운영 `.env`의 `TESTER_EMAIL`
- 비밀번호: 운영 `.env`의 `TESTER_PASSWORD`
- 공개 회원가입은 없습니다.

## 먼저 확인할 흐름

1. 프로젝트 목록에서 기존 프로젝트를 열거나 새 프로젝트를 만듭니다.
2. `쓰기` 화면에서 챕터를 만들고 원고를 작성합니다.
3. 저장 후 새로고침해도 원고와 에디터 설정이 유지되는지 봅니다.
4. Word 문서를 가져와 이어 쓰기가 가능한지 확인합니다.
5. 왼쪽 자료 창과 원고 창의 폭을 조절해 작업 배치가 편한지 봅니다.
6. 다크모드/라이트모드 전환이 읽기 편한지 확인합니다.
7. 음성 입력은 인식 원문과 정리본을 비교한 뒤 원고에 삽입합니다.
8. 원고 읽어주기는 선택 영역 또는 현재 챕터 전체로 테스트합니다.
9. 필요한 경우에만 `현재 챕터 분석` 또는 `스토리 분석하기`를 실행합니다.
10. 분석 리포트에서 `OpenAI 분석` 또는 `Mock 분석` 배지가 기대한 방식으로 보이는지 확인합니다.

## API 비용이 드는 동작

- `현재 챕터 분석`
- `전체 프로젝트 분석`
- 분석 리포트/프로젝트 화면의 `스토리 분석하기`
- 개발자용 `npm run analysis:test-openai`

원고 작성, 저장, 문서 가져오기, 음성 입력/읽어주기, 다크모드, 분할 창 조절은 OpenAI 비용이 들지 않습니다.

## 피드백 받을 질문

- 장시간 글쓰기 화면으로 편한가?
- 챕터, 자료 창, 분석 창의 분리가 이해되는가?
- AI 분석 결과가 원고 수정에 실제로 도움이 되는가?
- 캐릭터 설계와 원고 기반 분석이 헷갈리지 않는가?
- 음성 입력/읽어주기 기능이 퇴고에 도움이 되는가?
- 꼭 있어야 할 다음 기능은 무엇인가?

## 알려진 MVP 제한

- Web Speech API는 브라우저 지원 여부와 마이크 권한에 영향을 받습니다.
- SQLite는 소규모 MVP 테스트용입니다. 여러 사용자가 동시에 쓰는 운영 서비스로 커지면 PostgreSQL 전환이 필요합니다.
- OpenAI 분석은 비용 절약을 위해 기본적으로 챕터 단위 호출을 권장합니다.
